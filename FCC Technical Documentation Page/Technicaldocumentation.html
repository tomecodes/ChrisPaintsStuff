Introduction

HTML and CSS are the foundational technologies for building websites. HTML (HyperText Markup Language) is used to structure content on the web, while CSS (Cascading Style Sheets) is used to control the presentation and layout.

Whether you’re designing a simple personal portfolio or a complex web app interface, understanding how HTML and CSS work together is crucial. This documentation covers the essentials to get you started.

HTML Structure

An HTML document starts with a <!DOCTYPE html> declaration, which tells the browser the document type. The <html> element wraps everything, followed by a <head> and a <body> section. The <head> contains metadata like the page title and links to stylesheets. The <body> contains all the visible content of the webpage such as text, images, and links.

Common HTML Elements

HTML uses tags to define different types of content. Some of the most common elements include:

  • <h1> to <h6> for headings
  • <p> for paragraphs
  • <a> for links
  • <img> for images
  • <ul>, <ol>, and <li> for lists

<div> and <span>: Generic containers for structuring content. These tags allow you to organize information semantically and present it effectively.

CSS Syntax & Selectors

CSS is used to style HTML elements using rules made of selectors and declarations. A selector targets an HTML element, and declarations define what style to apply. For example:

p { color: blue; font-size: 16px; }

You can use element selectors (p), class selectors (.example), ID selectors (#main), and more complex combinations to target the exact parts of your page you want to style.

Box Model

Every HTML element is treated as a rectangular box in CSS, following the box model. This model includes:

  • Content: The actual text or image.
  • Padding: Space between content and border.
  • Border: The edge surrounding the padding.
  • Margin: Space outside the border that separates it from other elements. Understanding the box model is essential for layout and spacing on a web page.

Responsive Design

Responsive design ensures that your website looks good on all screen sizes—from phones to desktops. This is typically done using:

  • Relative units like percentages and em instead of fixed pixels.
  • CSS media queries (e.g., @media (max-width: 768px)) to apply styles based on screen size.
  • Flexible layouts using Flexbox or CSS Grid. Responsive web design improves usability and accessibility for all users.
